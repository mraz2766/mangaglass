# ExecPlan：MangaGlass UI 视觉聚焦刷新

## 摘要

在不改变解析、下载、持久化和队列行为的前提下，刷新 MangaGlass 的 SwiftUI 界面。重点提升视觉层级、共享样式一致性、章节选择可读性、下载状态密度，以及下载管理页的信息呈现。

本次属于低风险 UI 聚焦刷新，不做深度信息架构重写，不改 `MainViewModel` 的对外接口，也不触碰网络、解析和下载业务逻辑。

## 用户价值

- 用户打开应用后，可以更快识别主操作：输入漫画链接并加载。
- 批量下载前的分类和章节选择更容易扫读、选择和确认。
- 下载状态保持可见，但不再压过主工作区。
- UI 样式收敛到共享设计系统，后续继续优化界面时更安全、更一致。

## 范围

范围内：

- 新增 `Sources/MangaGlass/UI/DesignSystem.swift`，集中管理颜色、字体、间距、面板、状态 pill 和按钮样式。
- 刷新主界面顶部工具栏、左侧漫画 Inspector、右侧章节工作区、底部下载状态栏和下载管理页样式。
- 保留现有用户工作流和所有关键控件。
- 实施后运行 `swift build` 验证编译。

范围外：

- 不修改网络、解析器、下载队列、缓存、历史记录和持久化行为。
- 不新增第三方依赖。
- 不更新 README 截图。
- 不做深度业务状态拆分或信息架构重写。

## 约束

- 创建本计划时，项目内 `.agent/PLANS.md` 和全局 `/Users/mraz/.codex/PLANS.md` 均不存在，因此本计划最初沿用了 `docs/execplans/readme-bilingual.md` 的结构。
- 本次后续已新增 `.agent/PLANS.md`，并要求后续生成的 ExecPlan Markdown 正文默认使用中文。
- 实现必须保持低风险，改动集中在 SwiftUI 表达层。
- 应用通过 Swift Package Manager 构建，目标平台是 macOS 13，工具链为 Swift 6.2。

## 修改文件

- `Sources/MangaGlass/UI/DesignSystem.swift`
- `Sources/MangaGlass/UI/ContentView.swift`
- `Sources/MangaGlass/UI/DownloadManagerView.swift`
- `docs/execplans/ui-visual-refresh.md`

## 已确认事实

- 主界面外壳位于 `ContentView.swift`。
- 下载队列弹窗位于 `DownloadManagerView.swift`。
- 原有按钮、面板、颜色和字体样式散落在多个 UI 文件中。
- 现有截图和代码显示界面偏“玻璃卡片 + 胶囊 + 小字号密集控件”，视觉重点较平均。
- 业务状态由 `MainViewModel` 暴露，本次 UI 刷新不需要修改 ViewModel API。

## 实施计划

### 里程碑 1：创建共享视觉基础

- 新增 `DesignSystem.swift`，提供主题色、状态色、字体、间距、面板修饰器和统一按钮样式。
- 用共享设计系统替代本地重复的卡片、玻璃面板和按钮样式。

### 里程碑 2：刷新主工作区

- 重排工具栏，让 URL 输入框和“加载”成为最清晰的第一行动。
- 将站点入口、下载目录、解析状态和更多操作收敛到更安静的第二行。
- 降低左侧 Inspector 和章节面板中的嵌套卡片感、阴影和装饰性背景动效。

### 里程碑 3：改进章节选择与下载状态

- 用统一 accent 色、勾选图标和更接近列表的密度提升分类 chip 与章节 chip 的可读性。
- 保留拖拽框选、多选、排序、全选、清空和加入队列行为。
- 将底部下载摘要改为紧凑状态栏，突出队列状态、进度、速度、当前任务和下载管理/日志入口。

### 里程碑 4：刷新下载管理页

- 将筛选控件视觉收敛为更统一的 filter bar。
- 降低任务行阴影、圆角和卡片重量。
- 保留失败原因展开、队列控制、进度底栏和菜单操作。

### 里程碑 5：验证

- 运行 `swift build`。
- 检查 UI 改动是否误触 ViewModel 或服务层。
- 记录实际结果和意外发现。

## 验证方式

- `swift build` 必须通过。
- 应用仍能编译 `ContentView`、`DownloadManagerView` 和共享 UI helper。
- 主工具栏仍保留 URL 输入、加载、站点入口、下载目录、解析状态和更多菜单。
- 章节面板仍保留分类选择、排序、全选、清空、加入队列、点击选择和拖拽框选。
- 下载管理页仍保留筛选、暂停/继续、清空完成、重试失败、取消、清空队列、失败详情和进度底栏。

## 风险

- SwiftUI 大型 `ViewBuilder` 重排后可能出现类型推断问题。
- 抽取共享样式时可能与原有本地 modifier 或类型命名冲突。
- 视觉简化必须避免误删用户依赖的操作入口。

## 决策记录

- 决策：执行聚焦视觉刷新，而不是深度改版。
  原因：用户选择了低风险、见效快的改动范围。

- 决策：新增小型内部设计系统文件。
  原因：可以减少重复样式常量，同时不改变业务行为。

- 决策：保持 `MainViewModel` API 不变。
  原因：本任务聚焦表现层，不应影响解析和下载逻辑。

- 决策：新增 `.agent/PLANS.md` 并要求后续 ExecPlan 使用中文正文。
  原因：用户要求修改默认 plan，并要求生成的 Markdown 用中文内容填写。
