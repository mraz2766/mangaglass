# ExecPlan 默认模板

本文件是 MangaGlass 项目的默认 ExecPlan 模板。凡是按照 `AGENTS.md` 规则需要创建 ExecPlan 的复杂任务，都必须优先使用本模板，并将计划保存到 `docs/execplans/<task-name>.md`。

## 语言要求

- 生成的 ExecPlan Markdown 内容必须使用中文填写。
- 标题、章节名、说明、决策、进度、验证结果、回顾内容都应使用中文。
- 代码符号、文件路径、命令、类型名、函数名、错误信息和必要的英文专有名词可以保留原文。
- 如果引用用户原始英文需求，可以简短保留原文，但必须配中文解释。

## 文件命名

- 文件路径：`docs/execplans/<task-name>.md`
- `<task-name>` 使用简短英文 kebab-case，便于命令行检索。
- 文件内容使用中文，不因为文件名是英文而改用英文正文。

## 推荐结构

```markdown
# ExecPlan：<中文任务标题>

## 摘要

用 1-3 段说明要完成什么、为什么做、保持哪些行为不变。

## 用户价值

- 说明用户能感知到的改进。
- 避免只写代码结构。

## 范围

范围内：

- 列出本次明确要做的行为或文件级改动。

范围外：

- 列出不会做的内容，避免实现时扩大范围。

## 约束

- 写明技术、兼容性、项目规则、风险边界。
- 如果模板或上下文缺失，记录采用的默认决策。

## 修改文件

- `path/to/file`
- `docs/execplans/<task-name>.md`

## 已确认事实

- 记录通过阅读代码、配置、文档或命令确认的事实。
- 不要把未验证猜测写成事实。

## 实施计划

### 里程碑 1：<名称>

- 写清楚要改什么，以及完成后的可观察结果。

### 里程碑 2：<名称>

- 继续按行为或子系统分组。

## 验证方式

- 列出必须运行的命令。
- 列出用户可以观察到的验收场景。

## 风险

- 记录可能失败或需要注意的地方。

## 决策记录

- 决策：<做法>
  原因：<为什么这样做>

## 进度

- [ ] 创建并保存 ExecPlan。
- [ ] 完成主要实现。
- [ ] 完成验证。

## 意外发现

- 实施过程中发现的新事实、偏差或限制。

## 结果与回顾

- 完成后记录实际结果、验证结果、剩余风险。
```

## 执行要求

- 创建 ExecPlan 后，必须随着实施更新 `进度`、`决策记录`、`意外发现`、`结果与回顾`。
- 如果实施中偏离原计划，必须在 `决策记录` 中说明原因。
- 验证命令的结果必须写入 `结果与回顾`。
- 不要只在对话中维护计划，必须保存为持久 Markdown 文件。
